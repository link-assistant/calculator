### Fixed
- Fix CI/CD Auto Release job failing with "403 Forbidden" when `use-m` dynamically loads `command-stream` from npm registry
- Pin `command-stream` and `lino-arguments` to specific versions in all scripts to avoid `npm show` registry calls that can trigger 403 errors in GitHub Actions CI environment
- Root cause: `use-m` library always calls `npm show <package> version` for `@latest` packages (no caching), which fails with 403 in certain CI environments after npm global installs modify the registry state
